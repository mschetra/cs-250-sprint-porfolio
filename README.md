# CS 250 Agile Portfolio

This repository showcases my Agile development work for CS 250 at Southern New Hampshire University. It includes sprint artifacts, reflections, and documentation that demonstrate my understanding of Agile principles and how they support effective software development.

Throughout the course, I worked with Scrum concepts such as user stories, product backlog refinement, sprint planning, daily standups, and retrospectives. These artifacts highlight how iterative development, collaboration, and continuous improvement contribute to delivering high‑quality software.

## Contents
- **Sprint Review** – Summary of completed work, challenges, and outcomes.
- **Sprint Retrospective** – Reflection on team processes and opportunities for improvement.
- **Agile Documentation** – Notes on Scrum roles, ceremonies, and best practices.
- **Course Reflections** – Insights on how Agile methods influence real‑world software development.

## Purpose
This portfolio demonstrates my ability to apply Agile methodologies in a structured development environment. It reflects my growth in planning, communication, adaptability, and delivering incremental value—skills essential for modern software engineering.
